## 3. Justificativa

Pequenos comércios têm dados de PDV, mas não os transformam em decisão. O NEXO oferece uma análise orientada por dados acessível a esse público, contribuindo para uma gestão comercial mais informada.

## 4. Escopo

O escopo do MVP está definido no [Documento de Visão](documento_de_visao.md). Resumidamente: autenticação por perfis, cadastro de empresa/cliente, controle de plano, criação de análise por período, upload e processamento de relatórios, geração de indicadores, dashboard e devolutiva estratégica.

## 6. Ferramentas

- Versionamento: GitHub.
- Acompanhamento de backlog/tarefas: GitHub Projects ou Trello.
- Desenvolvimento: Python, Flask, SQLite, SQLAlchemy, Pandas, Plotly.js, Bootstrap, Jinja2.
- Prototipação: ferramenta a confirmar pela equipe (pendente de inclusão).

## 7. Estratégia de desenvolvimento

Desenvolvimento incremental em sprints, com cerimônias de planejamento, daily scrum e revisão. As funcionalidades são priorizadas pelo Product Backlog e selecionadas por sprint no Sprint Backlog.
