# Auditoria Final do GitHub - NEXO PI I

## Objetivo

Registrar a conferência final do repositório antes da entrega do **Projeto Integrador I**.

## Itens conferidos

| Item | Situação | Observação |
|---|---|---|
| README principal | Revisado | Descreve problema, solução, escopo, protótipo, tecnologias usadas e evidências. |
| Documentação-base | Organizada | Documentos distribuídos por etapas do PI I. |
| Design Thinking | Documentado | Mapa da Empatia, Persona/Jornada e HMW. |
| Requisitos | Documentados | RFs/RNFs e escopo do MVP. |
| Backlog | Documentado | Product Backlog, Sprint Backlog, Issues e lista de Issues pendentes. |
| Sprints | Documentadas | Planejamento e modelos de acompanhamento preparados. |
| Protótipo | Criado | Protótipo navegável em HTML/CSS/JavaScript. |
| Testes | Estrutura criada | Casos de teste e roteiro de usabilidade disponíveis. |
| Feedback e Refino | Estrutura criada | Preenchimento real depende da aplicação do teste. |
| Apresentação/Demo | Roteiros criados | Demo pode ser feita ao vivo com o protótipo. |
| Dados sensíveis | Protegidos | Termos assinados e dados pessoais não devem ir ao GitHub público. |

## Observação final

O repositório está organizado como evidência acadêmica do ciclo completo do Projeto Integrador I. Itens que dependem de execução real da equipe permanecem identificados sem invenção de dados.
