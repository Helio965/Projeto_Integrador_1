# Pendências Humanas da Entrega

Este documento lista itens que dependem de ação, confirmação ou informação real da equipe. Esses dados não devem ser inventados na documentação.

> Para a visão completa das pendências documentais, de GitHub, protótipo, validação e dados sensíveis, ver [pendencias_reais.md](pendencias_reais.md).

| Item | Pendência | Onde atualizar |
|---|---|---|
| Período letivo e datas reais | Confirmar período, datas de sprint, teste, demo e entrega oficial. | README.md, plano de projeto e documentos de sprint |
| GitHub Projects/Trello | Inserir link real do quadro usado pela equipe, caso seja criado além das Issues. | `docs/00_entrega_final/links_entrega.md` e `docs/04_backlog/github_projects_manual_setup.md` |
| Testes funcionais | Executar os testes funcionais reais e preencher resultados obtidos/status. | `docs/06_testes_usabilidade/plano_registro_testes_funcionais.md` |
| Teste de usabilidade | Registrar participante sem dados sensíveis, data, ambiente, duração e feedback real coletado. | `docs/06_testes_usabilidade/relatorio_feedback_usuario.md` |
| Refino da solução | Registrar ajustes reais feitos após o feedback. | `docs/07_refino_solucao/documento_refino_solucao.md` |
| Daily Scrum e Sprints | Preencher registros reais das cerimônias e da execução. | `docs/05_sprints/daily_scrum.md`, `sprint1_acompanhamento.md` e `sprint2_documento_tecnico.md` |
| Demo do MVP | Confirmar se a demo será ao vivo ou gravada; se houver vídeo, inserir link real. | `demo/README.md` e `docs/00_entrega_final/links_entrega.md` |
| Apresentação final | Incluir arquivo final em HTML, PDF ou slides, se exigido, ou anexar no canal oficial. | `apresentacao/` e AVA/Sala Online |
| Documentos oficiais (.docx) | Incluir em `docs/documentos_word/` quando permitido ou anexar pelo canal oficial. | `docs/documentos_word/` e AVA/Sala Online |
| Evidências permitidas | Inserir links ou arquivos permitidos, sem dados sensíveis. | `docs/01_visao_escopo/evidencias.md` e `evidencias/` |
| Termo de anuência | Manter arquivo assinado fora do GitHub público e enviar pela Sala Online, se exigido. | `docs/01_visao_escopo/termo_de_anuencia_orientacao.md` |
