# Plano de Labels e Milestones - NEXO

Este documento orienta a organização das Issues do GitHub para apoiar o acompanhamento do backlog do Projeto Integrador I.

## Labels recomendadas

| Label | Uso sugerido |
|---|---|
| `documentacao` | Documentos, roteiros, índices e evidências. |
| `design-thinking` | Mapa da empatia, persona, jornada e ideação. |
| `requisitos` | Requisitos funcionais, não funcionais e escopo. |
| `backlog` | Histórias de usuário, tarefas e priorização. |
| `sprint-1` | Itens planejados para a Sprint 1. |
| `sprint-2` | Itens planejados para a Sprint 2. |
| `teste` | Testes funcionais, usabilidade e validação. |
| `refino` | Ajustes derivados de feedback e melhoria da solução. |
| `prototipo` | Telas, navegação e demonstração do MVP. |
| `apresentacao` | Pitch, demo e materiais finais. |
| `pendente` | Item ainda não iniciado ou dependente de informação real. |
| `concluido` | Item concluído com evidência real. |

## Milestones recomendadas

| Milestone | Objetivo |
|---|---|
| `Sprint 1 - Base navegável` | Login simulado, perfis, cadastros, criação de análise e navegação base. |
| `Sprint 2 - Incremento funcional` | Upload simulado, processamento, indicadores, dashboard, devolutiva, publicação e visão CLIENTE. |
| `Validação e Refino` | Testes funcionais, teste de usabilidade, feedback e ajustes. |
| `Entrega Final` | Revisão final, demo, apresentação, evidências e envio oficial. |

## Observação

As labels e milestones devem ser criadas no GitHub quando a equipe tiver acesso ao repositório e confirmar a organização desejada. Este arquivo não substitui o quadro do GitHub Projects/Trello, mas serve como guia para padronizar o acompanhamento.
